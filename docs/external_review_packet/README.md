# External Review Packet Index

Purpose

- Separate the outward review packet from the internal operating files that live in this public folder.
- Keep the repo auditable without implying that every file here should be sent to a reviewer.

Reading rule

- If you are preparing or sending a first-pass external review request, use only the outward packet files listed below.
- If you are inspecting project process or coordination state, the internal operating files below are relevant, but they are not part of the first-pass packet.

Outward packet files

- `SYSTEM_SUMMARY.md`
- `PROOF_STATUS_SUMMARY.md`
- `ARCHITECTURE_SUMMARY.md`
- `DAIRY_SUBSYSTEM_BRIEF.md`
- `ASSUMPTIONS_SHEET.md`
- `QUESTION_SHEET.md`
- `REVIEW_RESPONSE_TEMPLATE.md`

Supporting outward requirements file

- `DAIRY_FIRST_EXTERNAL_REVIEW_BRIEF.md`: defines what the review is trying to establish and which reviewer disciplines are needed.

Internal operating files

- `INTERNAL_REVIEW_COORDINATION.md`
- `REVIEWER_TARGET_PIPELINE.md`
- `OUTREACH_MESSAGE_TEMPLATES.md`
- `REVIEW_SEND_LOG.md`
- `REVIEW_INTAKE_CONSOLIDATION_TEMPLATE.md`
- `CLAIM_RISK_MATRIX.md`
- `received_reports/`

Public disclosure rule

- Public visibility of this folder does not mean external review has been completed.
- Review completion claims should be based only on actual sent packets and actual attributable responses, not on the existence of these materials.
