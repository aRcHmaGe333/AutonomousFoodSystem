# Dairy-First Claim Boundaries

Status: Active

Purpose

- Keep outward language aligned with the actual proof surface.
- Prevent internal design intent from being mistaken for validated physical performance.
- Translate the current critique into reusable wording rules.

## Overarching safe statement

Use this when the repo needs one short livestock summary:

This dairy-first concept is partially modeled and technically interesting.
It is not yet physically validated.
Performance, welfare, sanitation, waste-loop, and economics claims remain hypotheses pending named external review and pilot data.

## Core language rules

- If a claim has no bounded tolerance, protocol, or measurement plan, describe it as a hypothesis.
- If a claim is supported only by simulation or concept logic, do not phrase it as operational proof.
- If a claim has been pressure-tested only by cited generic research or model-generated critique, do not call it external validation.
- If a reviewer objection would materially change investor interpretation, keep the claim softened until that objection is resolved.

## Phrases to avoid and safer replacements

| Avoid | Use instead |
|---|---|
| passive geometry materially reduces sensing complexity | the concept explores whether corridor geometry and indexing can simplify final positioning, subject to pilot validation |
| voluntary entry is likely | the concept aims for low-resistance entry, but real behavior remains a pilot question |
| lower-stress handling | lower-stress handling is a design intention, not a validated outcome |
| clean-zone separation is meaningful in practice | the design includes zone separation and sanitation logic that still require specialist review and pilot verification |
| digestate becomes a meaningful nutrient-loop output | the concept intends nutrient recovery from digestion, but usefulness still depends on mass-balance and digestate characterization |
| intervention rates will be low enough to matter economically | intervention rate and economics remain open questions pending pilot-envelope data |
| economically interesting design | potentially interesting if pilot data supports the operating envelope |

## What can be said safely today

- the repo contains a real software prototype
- the dairy-first subsystem is conceptually structured and simulation-guided
- the concept has explicit proof-status boundaries
- the project has a real external-review packet and validation discipline
- the current next step is targeted external critique plus narrow pilot planning

## What should not be said today

- that the livestock subsystem is physically validated
- that welfare outcomes are improved in practice
- that sanitation is solved
- that the waste loop is operationally proven
- that low-intervention economics are already credible
- that generic or synthetic reports count as completed external expert review

## Upgrade rule

Only strengthen a dairy-first outward claim when at least one of the following is true:

- the repo has direct implemented evidence for that claim
- the claim has survived named external expert review in the relevant discipline
- the claim has pilot data that matches the wording being used
