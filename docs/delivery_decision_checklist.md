# Delivery decision checklist — quick, plain guide

Purpose
-------
Help decide how to deliver goods for a pilot site. This guide keeps the question simple: what makes the most sense for the local conditions and the households served?

How to use (three quick steps)
-----------------------------
1. Gather simple facts about the site:
	- People density nearby (people per km²)
	- Typical delivery distance to customers (average km)
	- How close is a depot or hub (minutes)
	- Road access and width limits (yes/no)
	- How urgent deliveries must be (minutes, hours)
	- Typical package size and shape (standard crate vs odd shapes)
2. Compare the facts to the short rules below.
3. Use the result as a recommendation to test in a small pilot. Try the easiest option first and measure.

Practical rules (household-focused)
----------------------------------
- If density is very high (many people close together) and buildings are tall or connected, consider short fixed links (conveyor/pneumatic or local shafts). These work best when items are standard sizes and routes are short.
- If a depot is within ~15 minutes and road access is good, use vehicles or vans with micro-depots for last-mile runs. This is the most flexible option for mixed item types.
- If most deliveries are small, frequent, and within walking zones (e.g., pedestrian neighbourhoods), consider small robots for last‑meter delivery.
- If you control a campus or a clustered site (multiple nearby buildings under the same operator), conveyors or short automated links are practical for depot ↔ building moves.
- For urgent, tiny items (samples, secure documents), pneumatic tubes are useful inside buildings or between adjacent structures where carriers fit.

Simple thresholds (rule of thumb)
--------------------------------
- Density > 25,000 people/km² and standard packages → fixed short links (conveyor/pneumatic/local shaft).
- Depot travel time < 15 minutes and mixed package shapes → vehicles + micro-depot.
- Average delivery distance < 2 km and pedestrian zones → robots for last‑meter.

Mixed-mode recommendation
------------------------
Most real sites will use more than one method. Start with the depot model (vehicles + micro-depot) and add conveyor/pneumatic links or robots where they clearly reduce travel or latency.

What to test in a pilot
-----------------------
- Load and unload times for the chosen packaging.
- Depot wash/CIP throughput (can the depot clean returned containers fast enough?).
- Travel times and on‑street constraints at peak hours.
- Customer experience: time to unpack, reheat, and return containers.

Next steps (practical)
----------------------

- Alternatively, a one-page pilot checklist suitable for local ops teams would be useful.
